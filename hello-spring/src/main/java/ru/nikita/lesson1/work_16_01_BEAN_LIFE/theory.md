### Жизненный цикл бина

```
┌─────────────────────────────────────────────────────────────────┐
│                     ЖИЗНЕННЫЙ ЦИКЛ БИНА                        │
├─────────────────────────────────────────────────────────────────┤
│  1. Создание экземпляра (new)                                   │
│  2. Внедрение зависимостей (DI)                                │
│  3. Вызов @PostConstruct                                        │
│  4. ═══════════ БИН ГОТОВ К РАБОТЕ ═══════════                 │
│  5. ... бин используется приложением ...                       │
│  6. Вызов @PreDestroy (при остановке приложения)               │
│  7. Уничтожение бина                                           │
└─────────────────────────────────────────────────────────────────┘
```


### Bean Scopes (области видимости)

**Scope** определяет, сколько экземпляров бина создаётся и как долго они живут.

| Scope | Описание | Когда использовать |
|-------|----------|-------------------|
| `singleton` | Один экземпляр на весь контейнер (по умолчанию) | Почти всегда |
| `prototype` | Новый экземпляр при каждом запросе | Stateful объекты |
| `request` | Один экземпляр на HTTP-запрос | Веб-приложения |
| `session` | Один экземпляр на HTTP-сессию | Веб-приложения |
